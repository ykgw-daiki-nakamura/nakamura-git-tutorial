# 概要・基本方針

本セクションは、この製品の開発規約をまとめる。

## 目的と基本方針

本規約は、SaaS 版とセルフホスト版を単一コードベース・**同一バージョン**で提供するための運用を定める。対象は、ブランチ運用とリリース・デプロイ、そこへ流入する依存とビルド経路の防御、出荷後の障害対応、および顧客別カスタマイズの進め方である。

基本方針は次のとおり。

1. **trunk-based development**: `main` を唯一の統合ブランチとし、long-lived な開発ブランチを作らない。
2. **タグ起点のリリース**: SaaS 版・セルフホスト版ともに、`release/vX.Y` ブランチ上のタグ（= GitHub Release）からビルドされた成果物を出荷する。SaaS 版も `main` から継続デプロイはせず、リリース単位でバージョンアップする。
3. **build once, deploy many**: 出荷する成果物は RC タグから 1 度だけビルドし、GA では再ビルドせずに昇格させる。環境ごと・配布形態ごとのビルドもしない。検証した成果物と出荷する成果物が同一であることを、担保できる形に保つため。
4. **ブランチはバージョン系統、環境はデプロイ状態**: 環境（dev / staging / production）は **GitHub Environments とデプロイパイプライン**で表現する。環境ブランチは作成しない。
5. **upstream first**: すべての修正はまず `main` に取り込み、必要に応じて `release/*` へ cherry-pick する。逆方向のマージは禁止する。
6. **サプライチェーンは多層で守る**: 依存の取り込みを遅らせ、インストール時のコード実行を許さず、既知の脆弱性を CI で検知する。どの層も単独では破られる前提に立つ。
