# .claude/templates — 持ち運び可能なテンプレート

他プロジェクトへコピーして使える汎用テンプレート。プロジェクト固有の値は `<...>` を
置き換えて使う。

## 一覧

| ファイル | 用途 |
| --- | --- |
| [CLAUDE.md.template](CLAUDE.md.template) | 新規リポジトリ用の `CLAUDE.md` 雛形（概要／コマンド／規約／開発フロー／安全ガード／レビュー運用） |

## CLAUDE.md.template の使い方

1. コピーしてリポジトリ直下に置く。

   ```bash
   cp .claude/templates/CLAUDE.md.template ./CLAUDE.md
   ```

2. `<...>` のプレースホルダを自プロジェクトの内容に置き換える。
3. 不要な節は削り、プロジェクト固有の規約（CI・デプロイ・ディレクトリ運用）を足す。

## プロジェクト単位 vs 全体（`~/.claude/CLAUDE.md`）

CLAUDE.md は 2 層で考えると重複を避けやすい。

- **リポジトリの `CLAUDE.md`**: そのプロジェクト固有の情報（技術スタック・コマンド・
  ディレクトリ・固有規約）。本テンプレートが対象。
- **個人の `~/.claude/CLAUDE.md`**: どのプロジェクトでも共通の**自分の作法**。プロジェクトに
  依存しない普遍的な方針だけを書く。リポジトリ側と重複させない。推奨される内容の例:
  - コミットは Conventional Commits。着手前に計画を Issue 化し、PR に `Closes #N` を書く。
  - 破壊的・外向き操作は事前確認。シークレットをコミット／ログに載せない。
  - 変更後は該当プロジェクトの lint / build をローカルで通してから push する。
  - 迷ったら網羅より確実性を優先し、確認できた事実だけを断定する。

`~/.claude/CLAUDE.md` は個人領域のためこのリポジトリでは配布しない（各自で用意する）。

## レビュー支援（ビルトイン skill）

CLAUDE.md 雛形の「レビュー支援」節にあるとおり、レビュー系はビルトインを運用として呼ぶ
（再実装しない）。

- `/code-review` — 差分のバグ・簡素化レビュー
- `/simplify` — 差分の簡素化・重複排除の適用
- `/security-review` — 差分のセキュリティレビュー
