description: docs/guide に新しい章の雛形を生成し、config.mjs の sidebar 登録まで案内する
argument-hint: <slug> <章タイトル>
---

`docs/guide/<slug>.md` に新しいガイド章の雛形を作成し、サイドバー登録・検証まで一気通貫で行うコマンド。

## 入力

- `$1` … ファイル名 slug（拡張子なし・kebab-case。例 `stashing`）
- `$2` 以降 … 章タイトル（例 `一時退避 (stash) の使い方`）

slug かタイトルが未指定なら、何を作るかを 1 度だけ確認してから進める。

## 手順

1. `docs/guide/$1.md` が既に存在しないか確認する。あれば上書きせず中止し、その旨を報告する。
2. 下記「章テンプレート」を雛形として `docs/guide/$1.md` を新規作成する。見出しのタイトルは章タイトルで置き換え、本文セクションはプレースホルダのまま残す（執筆者が後で埋める）。
   - VitePress の内部リンクは**拡張子なし**で書く（例 `./branching`）。実ファイルは `.md`。
   - 図が必要なら Mermaid のコードフェンス（言語指定 `mermaid`）で追加する。日本語ラベルは `"..."` で囲むと崩れにくい。
   - コードフェンスの前後には必ず空行を入れる（markdownlint MD031）。
3. `docs/.vitepress/config.mjs` の `sidebar` に `{ text: '<章タイトル>', link: '/guide/$1' }` を追加する。内容に最も合うグループを選ぶ（既定の候補: 「はじめに・基礎」「チーム開発の基本フロー」「履歴とコンフリクトの扱い」「自動化とリリース」「発展」）。どのグループに入れるか迷う場合はユーザーに確認する。
4. 検証する。worktree/作業ツリー内で `npm run lint:md` と `npm run docs:build` を実行し、Markdown 整形エラー・内部リンク切れ・Mermaid 構文エラーが無いことを確認する。
5. 生成したファイルと sidebar 追記の差分を提示する。以降のコミット・PR は本リポジトリの GitHub Flow（着手前の Issue 化・`Closes #N`）に従う。

## 章テンプレート

以下を `docs/guide/$1.md` の初期内容とする（`<章タイトル>` と各プレースホルダは置換／加筆する）。

```text
# <章タイトル>

<この章で学ぶことと、読者が得られる成果を 2-3 文で書く>

## 前提

- <先に理解しておくとよいこと。関連する章へは拡張子なしリンクで誘導する>

## <本文セクション 1 の見出し>

<解説を書く。必要に応じて表・箇条書き・コード例を使う>

## <本文セクション 2 の見出し>

<解説を書く。フローや分岐は Mermaid 図で補足するとよい（フェンス前後は空行）>

## まとめ

- <この章の要点を 2-4 個の箇条書きで>

## 次のステップ

- <次に読むとよい章への拡張子なしリンク>
```
